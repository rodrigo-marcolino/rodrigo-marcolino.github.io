Problem Solving

By Rodrigo Marcolino | Posted on: 15 July 2021

In this sprint we learned a lot about problem solving, we saw some patterns to work on that and all were very helpful. I will describe my own method that I am developing from my journey here at Dev Academy.

In order to achieve that I will use one of our tasks from JavaScript Kata, the FizzBuzz, which is a classic programming exercise.

Understanding The Problem

The task consisted of two phases. In the first phase, was asked to write a program which prints out each number from 1 to 100. But for multiples of 3 print 'Fizz' instead of the number and for multiples of 5 print 'Buzz'. For numbers which are multiples of both 3 and 5 print 'FizzBuzz'.

In the second phase, was asked to Write a method called super_fizzbuzz which takes as its input an Array of integers.

It should loop over that array and then return a "fizzbuzzed" Array, i.e., the array is identical to the original, but with the following changes:

  1. Multiples of 3 should be replaced with the string "Fizz"
  2. Multiples of 5 should be replaced with the string "Buzz"
  3. Multiples of 15 should be replaced with the string "FizzBuzz"

Writing Pseudocode

Before I jump into the solution usually I write pseudocode for the problem. Definitely, it helps me defines the structure of my code.

The algorithm in pseudo code for this problem:


When a user inputs a number
If the current number is divisible by 3 then print "Fizz"
If the current number is divisible by 5 then print "Buzz"
If the current number is divisible by 3 and 5 then print "FizzBuzz"
Otherwise print the current number

However. the second part was a different history. I really got stuck in this part, I had tried many different approaches, basically, I went through all this process below:

  • Pseudocode
  • Replace Pseudocode With Real Code
  • Reading error messages and google about
  • Exhaustive use of Console.log
  • Googling

I was almost giving up and asking for help, and then suddenly, one of the topics from the topic "code Readability" that we learned in this sprint came up in my mind, the concept of D.R.Y (Don't Repeat Yourself).

In the previous solution, I already wrote a function to solve the problem for just numbers but without an array. What I've done was use this function within the built-in Array.map() method which allows iterating over an array and modify its elements using as a callback function.

I have to admit that I was proud of myself because it was a clean, simple and elegant solution.
